# Calendar

Time as a first-class surface. The calendar shows external events alongside native ones, and tasks land on it where they belong.

## Sources

- **External accounts** — Google Calendar, iCloud, Outlook. Multiple accounts at once.
- **Native events** — created inside the app. Stay local unless you choose to push them to an external calendar.
- **Tasks** — scheduled or due tasks appear on the calendar, but stay tasks.

## Sync behavior

- **Read** is always on: external events appear immediately.
- **Write** is opt-in per calendar: native edits push back to the source.
- **Conflict** — external truth wins on fields the source owns (title, time, attendees). Local fields (linked notes, your own annotations) are preserved.

## What you can do with an event

- Open or auto-create a [meeting note](meetings.md) for it.
- Link it to a task, doc, person, incident.
- Add private annotations that never sync out.

## Views

- Day / week / month / agenda
- Filter by calendar source or attendee
- Overlay tasks (on/off)
